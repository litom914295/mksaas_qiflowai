## 一般问题

### Q1: 如何访问管理后台？

**A**: 访问 `https://yourdomain.com/admin` 并使用管理员账号登录。如果是首次登录，请联系系统管理员获取初始凭证。

### Q2: 忘记管理员密码怎么办？

**A**: 
1. 点击登录页面的"忘记密码"链接
2. 输入注册邮箱
3. 查收密码重置邮件
4. 按照邮件指引重置密码

如果无法接收邮件，请联系技术支持。

### Q3: 支持哪些浏览器？

**A**: 建议使用以下浏览器的最新版本：
- ✅ Chrome 90+
- ✅ Firefox 88+
- ✅ Safari 14+
- ✅ Edge 90+

### Q4: 多人可以同时使用管理后台吗？

**A**: 可以。系统支持多用户并发访问，每个用户的操作都会被记录到审计日志中。

### Q5: 数据多久刷新一次？

**A**: 
- **实时监控**: 每 5 秒自动更新
- **错误日志**: 实时更新
- **性能指标**: 每分钟聚合
- **日志数据**: 实时流式更新

## 监控功能

### Q6: 为什么监控数据显示不准确？

**A**: 可能的原因：
1. **时区问题**: 检查浏览器和服务器时区设置
2. **数据延迟**: 等待数据采集和聚合完成
3. **缓存问题**: 清除浏览器缓存并刷新

解决方案：
- 点击刷新按钮
- 检查系统时间设置
- 验证数据采集服务运行状态

### Q7: 如何查看特定时间段的数据？

**A**: 
1. 使用页面上方的时间选择器
2. 可选择预设时间范围：
   - 最近 1 小时
   - 最近 24 小时
   - 最近 7 天
   - 自定义范围
3. 点击"应用"查看数据

### Q8: 错误日志太多，如何快速定位问题？

**A**: 使用高级筛选功能：
1. 按**错误级别**筛选（Error/Critical）
2. 按**发生频率**排序
3. 使用**搜索功能**查找特定错误
4. 查看**错误趋势图**识别异常时间点

### Q9: 性能监控中的 P95、P99 是什么意思？

**A**: 
- **P95**: 95% 的请求响应时间低于此值
- **P99**: 99% 的请求响应时间低于此值

这些指标比平均值更能反映真实用户体验，尤其是慢请求的情况。

### Q10: 如何导出监控数据？

**A**: 
1. 设置所需的筛选条件和时间范围
2. 点击页面右上角的"导出"按钮
3. 选择导出格式（CSV 或 JSON）
4. 文件将自动下载到本地

## 告警系统

### Q11: 如何创建新的告警规则？

**A**: 
1. 进入 **监控中心 > 告警配置**
2. 点击"新建告警规则"
3. 填写规则信息：
   - 名称和描述
   - 触发条件
   - 持续时间
   - 严重程度
   - 通知渠道
4. 保存并启用规则

### Q12: 为什么设置了告警但没有收到通知？

**A**: 请检查：
1. **规则是否启用**: 确认告警规则开关为"开启"状态
2. **条件是否满足**: 查看当前指标值是否真正超过阈值
3. **持续时间**: 确认问题是否持续了设定的时间
4. **通知渠道**: 验证邮箱、Slack 等配置是否正确
5. **告警历史**: 查看告警是否被触发但发送失败

### Q13: 如何避免告警风暴（频繁告警）？

**A**: 
1. **调整阈值**: 根据实际情况设置合理的阈值
2. **增加持续时间**: 避免瞬时波动触发告警
3. **设置冷却期**: 同一告警间隔一定时间才重复发送
4. **告警聚合**: 将相似告警合并
5. **静默时段**: 设置维护窗口期

### Q14: 告警通知可以发送到哪些渠道？

**A**: 当前支持：
- ✉️ **邮件** - 支持多个收件人
- 💬 **Slack** - 指定频道或用户
- 📱 **短信** - 紧急告警
- 🔗 **Webhook** - 自定义集成

### Q15: 如何测试告警规则是否有效？

**A**: 
1. 在告警规则详情页
2. 点击"测试规则"按钮
3. 系统会模拟触发告警
4. 检查是否收到测试通知

## 备份恢复

### Q16: 备份存储在哪里？

**A**: 备份可以存储在：
- **云存储**: AWS S3、阿里云 OSS、腾讯云 COS
- **本地存储**: 服务器本地磁盘（不推荐）
- **网络存储**: NAS 或其他网络设备

配置路径：**系统配置 > 云存储配置**

### Q17: 备份多久执行一次？

**A**: 默认配置：
- **数据库完整备份**: 每天凌晨 2:00
- **增量备份**: 每 6 小时
- **日志备份**: 可选

可以在 **系统配置 > 定时任务** 中自定义备份计划。

### Q18: 备份文件保留多久？

**A**: 默认保留策略：
- **完整备份**: 保留 30 天
- **增量备份**: 保留 7 天
- **归档备份**: 可配置更长时间

超过保留期的备份会自动删除以节省空间。

### Q19: 如何验证备份是否可用？

**A**: 
1. 定期执行**备份测试恢复**（建议每月一次）
2. 步骤：
   - 在测试环境恢复备份
   - 验证数据完整性
   - 检查应用能否正常运行
3. 查看备份日志确认无错误

### Q20: 恢复数据需要多长时间？

**A**: 恢复时间取决于：
- 备份文件大小
- 网络速度（云存储）
- 服务器性能

参考时间：
- **小型数据库** (< 1GB): 5-10 分钟
- **中型数据库** (1-10GB): 30-60 分钟
- **大型数据库** (> 10GB): 1-3 小时

## 配置问题

### Q21: 如何配置 Sentry？

**A**: 
1. 在 Sentry.io 创建项目获取 DSN
2. 进入 **系统配置 > Sentry 配置**
3. 填写 DSN 和其他配置
4. 点击"测试连接"验证
5. 保存配置

详细步骤见[管理员手册](./ADMIN_USER_MANUAL.md#sentry-配置)。

### Q22: Cron 表达式怎么写？

**A**: Cron 表达式格式：`分 时 日 月 周`

常用示例：
```
0 2 * * *      # 每天凌晨 2:00
0 */6 * * *    # 每 6 小时
0 0 * * 0      # 每周日凌晨
0 0 1 * *      # 每月 1 号凌晨
*/30 * * * *   # 每 30 分钟
```

使用 [Crontab.guru](https://crontab.guru/) 辅助生成。

### Q23: 如何更新环境变量？

**A**: 
⚠️ **注意**: 修改环境变量需要重启服务才能生效！

1. 进入 **系统配置 > 部署管理**
2. 切换到"环境变量"标签
3. 编辑或添加变量
4. 保存更改
5. 触发重新部署

### Q24: 云存储凭证如何更新？

**A**: 
1. 进入 **系统配置 > 云存储配置**
2. 更新 Access Key 和 Secret
3. 点击"测试连接"验证新凭证
4. 保存配置

**最佳实践**: 定期轮换访问密钥（建议每90天）。

### Q25: 支持多环境配置吗？

**A**: 支持。系统可以配置：
- **开发环境** (Development)
- **测试环境** (Staging)
- **生产环境** (Production)

每个环境使用独立的配置文件和数据库。

## 性能问题

### Q26: 监控页面加载很慢怎么办？

**A**: 优化建议：
1. **缩小时间范围**: 查询较短时间段的数据
2. **使用筛选器**: 减少数据量
3. **清除缓存**: 清理浏览器缓存
4. **网络检查**: 确认网络连接正常
5. **分页加载**: 使用分页功能

### Q27: 实时数据流断开怎么办？

**A**: 
1. **刷新页面**: 通常可以自动重连
2. **检查网络**: 确认网络稳定
3. **查看控制台**: 检查浏览器控制台错误
4. **手动重连**: 点击"重新连接"按钮

如果持续断开，请联系技术支持。

### Q28: 数据库查询超时？

**A**: 可能原因和解决方案：
1. **查询范围过大**: 缩小时间范围或添加筛选条件
2. **数据库负载高**: 等待一段时间后重试
3. **索引问题**: 联系技术支持优化查询

### Q29: 如何提高告警响应速度？

**A**: 
1. 使用**多通知渠道**确保及时收到
2. 设置**优先级**，关键告警立即通知
3. 配置**值班轮换**确保有人响应
4. 使用**告警聚合**减少噪音

### Q30: 系统支持多少并发用户？

**A**: 
- **推荐**: < 50 并发用户
- **最大**: < 100 并发用户

如需支持更多用户，需要升级服务器配置或使用负载均衡。

## 安全相关

### Q31: 如何确保管理后台安全？

**A**: 安全措施：
1. ✅ **强密码策略**: 密码至少 12 位，包含大小写字母、数字、符号
2. ✅ **定期更换密码**: 每 3 个月更换一次
3. ✅ **启用 2FA**: 双因素认证（如果支持）
4. ✅ **限制 IP**: 只允许特定 IP 访问
5. ✅ **审计日志**: 定期审查操作日志

### Q32: 数据传输是否加密？

**A**: 是的。
- ✅ 所有通信使用 **HTTPS/TLS** 加密
- ✅ 敏感数据在数据库中加密存储
- ✅ API 密钥和凭证加密保存

### Q33: 如何查看操作审计日志？

**A**: 
1. 进入 **系统管理 > 审计日志**
2. 可以查看：
   - 操作时间和操作者
   - 操作类型和详情
   - IP 地址和设备信息
3. 支持按用户、时间、操作类型筛选

### Q34: 发现可疑活动怎么办？

**A**: 立即采取行动：
1. **更改密码**: 立即更换可能泄露的密码
2. **撤销权限**: 暂时禁用可疑账户
3. **查看日志**: 审查审计日志确定影响范围
4. **联系支持**: 报告安全事件
5. **通知团队**: 告知相关人员

### Q35: 如何备份配置信息？

**A**: 
1. **自动备份**: 配置信息包含在数据库备份中
2. **手动导出**: 
   - 进入各配置页面
   - 点击"导出配置"
   - 保存到安全位置
3. **版本控制**: 重要配置使用版本控制管理
