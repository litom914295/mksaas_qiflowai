# 积分策略调整说明

## 📋 调整内容

### 之前的问题：
- ❌ **Pro 月订阅**：$9.90/月 → 1000积分（30天过期）
- ❌ **购买积分包（Enterprise）**：$69.90 → 1000积分（30天过期）
- ❌ **不合理**：订阅比直接购买便宜7倍，但获得相同积分和期限

### 调整后的方案：

#### 1. Pro 订阅（月付/年付）
- 价格：$9.90/月 或 $99/年
- 积分：每月 **300 积分**
- 有效期：**永久有效**（可累积）
- 优势：长期使用，积分不过期

#### 2. Lifetime 终身版
- 价格：$199（一次性）
- 积分：每月 **500 积分**
- 有效期：**永久有效**（可累积）
- 优势：终身使用，无需续费

#### 3. 购买积分包
- 价格：$9.90 - $69.90
- 积分：100 - 1000 积分
- 有效期：**30天**
- 优势：灵活购买，适合短期需求

## 🎯 新策略的优势

### 1. 合理的价值体系
✅ **订阅用户**：每月300积分（永久），长期累积价值更高  
✅ **购买积分**：1000积分（30天），适合短期大量使用  
✅ **价格合理**：订阅提供长期价值，积分包提供灵活性

### 2. 清晰的用户分层
- **长期用户** → 订阅（积分永久）
- **重度用户** → Lifetime（积分永久 + 更多）
- **偶尔用户** → 购买积分包（按需）

### 3. 鼓励订阅
- 订阅积分**永久有效**，鼓励用户长期使用
- 购买积分包有**时间限制**，适合临时需求
- 形成良性的商业模式

## 💡 实施说明

### 配置文件更改
文件：`src/config/website.ts`

```typescript
// Pro 订阅
credits: {
  enable: true,
  amount: 300,      // 从 1000 降到 300
  expireDays: 0,    // 从 30 改为 0（永久有效）
}

// Lifetime
credits: {
  enable: true,
  amount: 500,      // 从 1000 降到 500
  expireDays: 0,    // 从 30 改为 0（永久有效）
}
```

### 数据库影响
- ✅ 现有用户的积分**不受影响**
- ✅ 新订阅用户按新规则执行
- ✅ `expireDays: 0` 表示积分永久有效

## ⚠️ 注意事项

1. **现有订阅用户**：
   - 如果用户已经购买了旧规则的订阅（1000积分/月），需要决定是否：
     - 保持旧规则直到续费
     - 或者立即切换到新规则

2. **积分计算逻辑**：
   - 需要确认系统正确处理 `expireDays: 0` 的情况
   - 确保积分不会被自动过期清理

3. **用户沟通**：
   - 向用户说明新策略的优势
   - 强调订阅积分永久有效的价值
