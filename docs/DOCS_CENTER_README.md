## 访问地址

- **文档中心首页**: `/admin/docs`
- **文档详情页**: `/admin/docs/[doc-id]`

## 文档维护

### 添加新文档

1. 在 `docs/` 目录创建 Markdown 文件
2. 在 `src/config/docs.config.ts` 添加文档配置
3. 设置正确的分类、标签和元数据
4. 提交代码并测试

### 更新文档

1. 修改对应的 Markdown 文件
2. 更新 `lastUpdated` 时间戳
3. 如有需要，更新标签和描述

### 文档规范

- 使用 Markdown 格式编写
- 添加清晰的标题层级
- 提供代码示例和截图
- 包含相关链接
- 估算阅读时间

## 常见问题

### Q: 文档无法显示？
**A**: 检查文档文件路径是否正确，确保文件存在于 `docs/` 目录

### Q: 搜索找不到文档？
**A**: 检查文档配置中的标题、描述和标签是否包含搜索关键词

### Q: 如何添加新的文档分类？
**A**: 在 `docs.config.ts` 的 `DOC_CATEGORIES` 中添加新分类，并更新类型定义

### Q: 文档图片显示问题？
**A**: 图片应放在 `public/docs/` 目录，使用相对路径引用
