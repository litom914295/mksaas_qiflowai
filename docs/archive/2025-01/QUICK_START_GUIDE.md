# 🚀 快速启动和测试指南

## 重启开发服务器

```powershell
# 如果服务器正在运行，先停止 (Ctrl+C)

# 重新启动开发服务器
npm run dev
```

## 🧪 测试新功能

### 1. 测试密码强度指示器

访问注册页面：
```
http://localhost:3000/auth/register
```

**测试用例：**

#### 弱密码 🔴
输入: `abc`
- 预期：1个红色进度条
- 建议："至少使用 8 个字符"、"添加大写字母"

#### 一般密码 🟠
输入: `abcd1234`
- 预期：2个橙色进度条
- 建议："添加大写字母"

#### 良好密码 🟡
输入: `Abcd1234`
- 预期：3个黄色进度条
- 建议："添加特殊字符"

#### 强密码 🟢
输入: `Abcd1234!`
- 预期：4个绿色进度条
- 无建议

### 2. 测试错误消息

#### 重复邮箱测试
1. 使用已注册的邮箱尝试注册
2. 预期错误消息：
   - 🇬🇧 EN: "This email is already registered. Please use another email or try logging in."
   - 🇨🇳 ZH: "该邮箱已被注册，请使用其他邮箱或尝试登录"

#### 成功注册测试
1. 使用新邮箱和强密码
2. 填写姓名
3. 点击注册
4. 预期：显示成功消息 "Please check your email inbox"

### 3. 测试多语言切换

切换到中文：
```
http://localhost:3000/zh-CN/auth/register
```

验证：
- ✅ 密码强度标签显示为"密码强度"
- ✅ 强度等级显示为"弱/一般/良好/强"
- ✅ 建议显示为中文

## 🐛 故障排除

### 问题 1: 密码强度指示器不显示
**解决方案：**
```powershell
# 清除 Next.js 缓存
Remove-Item -Recurse -Force .next
npm run dev
```

### 问题 2: 翻译没有更新
**解决方案：**
```powershell
# 重启开发服务器
# Ctrl+C 停止，然后
npm run dev
```

### 问题 3: 仍然看到 500 错误
**检查清单：**
1. 验证 `.env` 文件编码为 UTF-8
2. 验证 `DATABASE_URL` 正确
3. 测试数据库连接：
```powershell
npx drizzle-kit push
```
