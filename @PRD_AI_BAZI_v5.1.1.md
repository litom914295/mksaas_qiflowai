# PRD: AI 八字风水首页 (v5.1.1)

## 1. 项目定位
**产品名称**: AI 八字风水智能平台（QiFlow AI）  
**目标用户**: 18岁以上，对中国传统文化（八字、风水）感兴趣，并希望借助 AI 技术获得个性化分析与咨询的用户。  
**核心价值**: 融合千年传统智慧与现代 AI 技术，提供精准八字分析、风水罗盘布局、AI 咨询等多维服务，助力用户在事业、健康、财运等方面获得科学决策支持。

## 2. 用户心理与诉求
### 2.1 用户画像
- **主要人群**: 25-50 岁城市白领、创业者、自由职业者
- **核心诉求**:
  - 了解自身命理特征，寻求事业/感情/健康指引
  - 风水布局调整，提升生活/办公环境能量场
  - 快速获得可信且个性化的建议，降低盲目试错成本
- **心理特征**:
  - 对传统文化有兴趣，但缺乏深度理解能力
  - 希望快速、专业、隐私的服务体验
  - 对 AI 辅助决策有信任感，但也需要透明的专业依据

### 2.2 用户痛点
- 线下咨询费用高、时间成本大
- 网上信息碎片化、不可信
- 传统算命界面陈旧，缺乏科技感与易用性
- 缺乏实时交互与深度个性化服务

## 4. UX 四态设计
### 4.1 Empty State（空态）
- **场景**: 用户首次访问，未进行任何操作
- **组件**: 空状态插图 + 引导文案（"立即开始您的命理之旅"）
- **验收**: 首页默认显示 CTA 按钮，清晰指引用户进入功能入口

### 4.2 Error State（错误态）
- **场景**: API 调用失败、网络异常、数据格式错误
- **组件**: 错误提示卡片（带重试按钮）
- **示例文案**: "哎呀，服务暂时不可用，请稍后重试"
- **验收**: 错误信息友好，不暴露技术细节；提供重试入口

### 4.3 Limited State（受限态）
- **场景**: 积分不足、账户未认证、功能未开放
- **组件**: 蒙层提示 + 充值/升级引导
- **示例文案**: "您的积分不足，充值后可继续使用"
- **验收**: 清晰告知限制原因，提供解决路径（充值/认证）

### 4.4 Timeout State（超时态）
- **场景**: AI 计算耗时过长（>10秒）
- **组件**: Loading 骨架屏 + 进度提示
- **示例文案**: "AI 正在分析您的命盘，请稍候..."
- **验收**: 超过 10 秒显示进度提示，超过 30 秒提供取消/重试选项

## 7. 技术实现要求
### 7.1 技术栈
- **框架**: Next.js 15 (App Router)
- **语言**: TypeScript
- **样式**: Tailwind CSS + Shadcn UI
- **国际化**: next-intl（支持中文/英文）
- **性能**: React Server Components (RSC) + Suspense + 动态加载

### 7.2 文件结构
```
qiflow-ai/src/app/[locale]/
  page.tsx                      # 首页入口（空文件，需创建）
  layout.tsx                    # 已有布局文件（Header + Footer）
  components/
    home/
      HeroSection.tsx           # 英雄区组件
      FeaturesSection.tsx       # 功能介绍区
      HowItWorks.tsx            # 如何运作
      Testimonials.tsx          # 用户见证
      PricingSection.tsx        # 积分计费说明
      TrustSection.tsx          # 信任与合规区
public/brand/
  logo.svg                      # 品牌 Logo
  icons/
    bagua.svg                   # 八卦图标
    luopan.svg                  # 罗盘图标
    ai.svg                      # AI 图标
    shield.svg                  # 安全/信任图标
    lightning.svg               # 快速/效率图标
```

### 7.3 代码规范
- 使用 `function` 关键字定义纯函数，使用箭头函数定义 React 组件
- 严格类型检查（TypeScript `type` 定义 props）
- 避免 `enum`，使用字符串字面量联合类型
- 所有图片使用 `next/image`，提供显式 `width` 和 `height`
- 客户端组件最小化，仅交互叶子节点使用 `'use client'`
